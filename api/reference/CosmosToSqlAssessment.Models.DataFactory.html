DataFactoryArtifact<TProps>

Generic ADF resource envelope: { "name": ..., "properties": { "type": ..., ... } }. The envelope is strongly typed; AdditionalProperties on the concrete Properties bag keeps everything extensible so later sub-issues (#142–#147) can layer parameters, retry policies, monitoring, etc. without breaking #141's shape.

DatasetProperties

Properties bag for an ADF Dataset artifact, extending PropertiesBase with the type discriminator, linked-service reference, and connector-specific type properties.

DatasetResource

ADF Dataset artifact. Stand-alone JSON file under ADF/Datasets/.

LinkedServiceProperties

Properties bag for an ADF Linked Service artifact, extending PropertiesBase with the type discriminator and connector-specific type properties such as connection strings, account endpoints, and authentication settings.

LinkedServiceResource

ADF Linked Service artifact. Stand-alone JSON file under ADF/LinkedServices/.

PipelineActivity

Pipeline activity. Activity type (e.g. Copy, ExecutePipeline) is the discriminator; activity-specific configuration lives under TypeProperties. Retry policy / on-failure / depends-on / linked-service refs are layered in by follow-on sub-issues via the extension bag.

PipelineProperties

Properties bag for an ADF Pipeline artifact, extending PropertiesBase with the ordered list of activities that define the pipeline's execution graph.

PipelineResource

ADF Pipeline artifact. Stand-alone JSON file under ADF/Pipelines/.

PropertiesBase

Base for the properties bag of every ADF artifact. Provides a free-form AdditionalProperties extension dictionary plus the universally applicable Annotations list.

ResourceReference

Reference to a sibling artifact by logical name. ADF uses this shape everywhere (datasets in inputs/outputs, linked services on datasets, pipelines in ExecutePipeline activities, etc.). The optional Parameters bag is used by callers that pass values into a parameterised dataset, linked service, or pipeline reference.
