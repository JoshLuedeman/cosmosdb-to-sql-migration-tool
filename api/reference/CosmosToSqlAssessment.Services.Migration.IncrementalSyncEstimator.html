Estimates and compares the initial bulk-load duration against change-feed incremental sync time and steady-state behavior (#135 of parent #69). Pure analysis over the already-collected CosmosDbAnalysis and DataFactoryEstimate; performs no live Cosmos DB calls and reuses the existing Data Factory initial-load estimate as its baseline.

Remarks

All figures are heuristic planning estimates, not guaranteed throughput. Incremental capacity is approximated as a configurable multiple of the estimated end-to-end initial-load throughput (default ×1.0), deliberately conservative because incremental sync typically performs more expensive target-side upserts/merges than the initial insert-only load.
