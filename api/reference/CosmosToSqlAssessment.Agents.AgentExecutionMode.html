Strategy the AgentOrchestrator uses to schedule a set of agents.

public enum AgentExecutionMode

Fields

Sequential = 0

Run agents one at a time in dependency order. Deterministic and easiest to debug.

Parallel = 1

Run agents whose dependencies are already satisfied concurrently, scheduling more as dependencies complete. Maximizes throughput for independent work.

Conditional = 2

Run agents in dependency order but skip any agent whose required upstream outputs are missing (for example because an upstream agent failed), rather than failing the whole run.

Remarks

The mode selects a scheduling policy; actual ordering and gating are derived from each agent's Dependencies so the orchestrator never hardcodes per-agent rules.
