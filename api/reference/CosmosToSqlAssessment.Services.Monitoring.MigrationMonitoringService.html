Remarks

The service consumes an IAsyncEnumerable<T> of MigrationProgressSample (the streaming convention from #129), derives per-sample custom metrics, publishes them through an IMigrationMetricPublisher, and yields a snapshot for each sample.

Cumulative totals and rates are accumulated per pipeline/run/activity key, so interleaved samples from concurrent runs never cross-contaminate. A publish failure is logged and swallowed by the publisher so it never aborts the stream.
