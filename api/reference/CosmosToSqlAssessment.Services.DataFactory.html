ActivityPolicyBuilder

Builds the ADF policy block for an activity (#143). Pure / side-effect free — returns a serialiser-ready dictionary which the orchestrator and copy-activity builder stash on AdditionalProperties.

AdfJsonSerializer

Single source of truth for ADF JSON serialization options. All artifact writers MUST route through this class so output is deterministic, camelCase, indented, and skips nulls — the shape Azure Data Factory imports/deploys cleanly.

AdfNameRegistry

Allocates collision-proof, ADF-compliant identifiers for activities, datasets, linked services, pipelines, and the on-disk file names that mirror them. ADF allows up to 260 chars for most resources but activity names are capped at 55. We use the tighter budget for everything so a single sanitizer covers all artifacts. Case-insensitive collisions get a stable 8-char hash suffix derived from the full source identity.

ArmParameterDefinition

ArmResourceInput

One ADF artifact slated for inclusion in the ARM template.

ArmTemplateBuilder

Builds the deployable Azure Resource Manager template that wraps every emitted ADF artifact (linked services, datasets, pipelines) under Microsoft.DataFactory/factories/{kind} resources (#146).

Design notes:

  • Serialise-then-parse input: every per-artifact properties bag is captured as a JsonObject (via JsonSerializer.SerializeToNode) so [JsonExtensionData] AdditionalProperties is already flattened — the ARM template payload matches what's written to the per-artifact JSON file byte-for-byte (modulo the targeted defaultValue rewrite below).
  • Generic reference walker: CollectReferences(JsonNode, List<(string Kind, string Name)>) recurses through every JsonObject /JsonArray in the properties bag looking for { "type": "*Reference", "referenceName": <name> } triples. This automatically catches Dataset / LinkedService / Pipeline references plus the Key Vault LinkedServiceReference nested under typeProperties.accountKey.store on the Cosmos/SQL linked services — no per-activity-type code needed.
  • Closed-world dependsOn: edges are only emitted for resources actually present in this template's resource set. External / operator-managed linked services (e.g. a fault-tolerance log storage LS the operator has pre-provisioned) are silently skipped here — the operator's existing deployment of those resources satisfies them at New-AzResourceGroupDeployment time.
  • Targeted defaultValue rewrite: when a pipeline's PipelineParameterArmOverrides map is non-null, the builder walks properties.parameters.<pipelineParamName>.defaultValue and replaces the literal with [parameters('<armParamName>')]. This is the only mechanism that actually flows deployment-time parameter values into deployed ADF pipelines — without it, the parameter file would be decorative.
  • Determinism: resources sorted by (kind, name) and dependsOn entries sorted alphabetically so the file diff is stable across runs.

CopyActivityPolicy

DataFactoryGenerationResult

DataFactoryPipelineGenerationService

Orchestrates Azure Data Factory artifact generation: composes the per-database linked service, all source / sink datasets, one Copy activity per ContainerMapping, and writes every artifact under <outputDirectory>/ADF/{LinkedServices,Datasets,Pipelines}/.

Pipelines are split per source database, and any database with more than MaxActivitiesPerPipeline mappings is chunked into multiple pipeline files so ADF's per-pipeline activity limit is respected. A master orchestrator pipeline references every per-database pipeline via ExecutePipeline activities and forwards per-database parameters down.

DatasetBuilder

Builds ADF source / sink datasets for every Cosmos container and every target SQL table. Datasets declare parameters (e.g. databaseName, collectionName) and forward them into the linked-service reference and typeProperties so the same dataset can be reused across environments by passing parameter values from the invoking pipeline (#142).

DiagnosticSettingsTemplateBuilder

Emits a stand-alone, deployable ARM template containing a Microsoft.Insights/diagnosticSettings resource scoped to the target Data Factory. Sets logAnalyticsDestinationType = "Dedicated" so logs land in the resource-specific ADFPipelineRun / ADFActivityRun tables (matching the bundled KQL queries) rather than the legacy AzureDiagnostics table.

ExecutePipelinePolicy

ADF policy shape applied to every emitted ExecutePipeline activity (#143).

FailureNotificationBuilder

Builds the Web + Fail activity pair that fires when an upstream activity fails (#143). The Web activity POSTs to a parameterised webhook (Logic App / Teams / PagerDuty / etc) with the failed activity's error message, and a subsequent Fail activity re-throws so the pipeline still reports failure to its caller. The Web activity has policy.secureInput/secureOutput set so the webhook URL never enters run history.

FaultToleranceOptions

Copy-activity fault tolerance options (#143).

IncrementalCopyActivityBuilder

Builds the per-mapping incremental activity group (#147). For each container → table mapping selected for incremental copy, the orchestrator inserts:

  1. LookupWatermark_<m> — Azure SQL Lookup with a literal-mapping-key SELECT against the watermark table.
  2. SetVariable_LastTs_<m> — Reads the Lookup output into lastWatermark_<m>.
  3. SetVariable_NewTs_<m> — Computes newWatermark_<m> = max(lastWatermark, unix(utcnow()) - safetyLag). The clamp prevents safety-lag arithmetic from producing a backwards window.
  4. (when validation on) LookupSrc_<c> with incremental WHERE.
  5. The Copy activity (passed in pre-built; this builder rewrites typeProperties.source.query).
  6. ScriptUpdateWatermark_<m> — Script (NonQuery) running the MERGE with backwards-protection guard, depends on Copy Succeeded.

Mapping key format: <srcDb>::<srcContainer>->[<tgtDb>].[<schema>].[<table>]. This composite key prevents same-named containers across different source databases (or fan-out to multiple sink tables) from corrupting each other's watermark — see rubber-duck Blocker B5.

IncrementalCopyOptions

Incremental copy configuration (#147). Default Enabled is false so output is byte-identical to the post-#146 baseline unless the operator opts in.

When enabled, each per-mapping Copy is wrapped in a LookupWatermark → SetVariable_LastTs → SetVariable_NewTs → [LookupSrc] → Copy → ScriptUpdateWatermark chain. The Copy's typeProperties.source.query is overridden with a WHERE c._ts > lastTs AND c._ts <= newTs predicate. The newTs is clamped to utcnow() - WatermarkSafetyLagSeconds to avoid losing documents committed within the boundary second (Cosmos _ts is second-granular).

Mapping Data Flow change-feed support (the second Microsoft-documented incremental pattern, exposed as enableChangeFeed / startFromBeginning on the data-flow source) is intentionally NOT in this sub-issue. The IncrementalCopyMode.ChangeFeedDataFlow enum slot is reserved for parent #69 to add later without breaking the option surface.

IncrementalGroup

Per-mapping activity group produced by BuildGroup(ContainerMapping, string, string, string, string, PipelineActivity, AdfNameRegistry, IncrementalCopyOptions). Names + variables surface to the orchestrator so the per-db pipeline can declare matching parameters, variables, and (optionally) a parameter-template entry.

JsonPathEscaper

Builds JSONPath expressions for ADF translator mappings. Bracket-notation is used for every segment so field names containing dots, hyphens, spaces, or quotes work without manual escaping by the operator.

LinkedServiceBuilder

Builds the parameterised ADF linked services. Default emit mode is System-Assigned Managed Identity for both connectors (the modern recommended shape). Key Vault wiring is opt-in via UseAzureKeyVault.

MonitoringOptions

Monitoring & logging configuration (#144). Defaults are deployment-safe: the diagnostic-settings ARM template is emitted, every Copy activity gets monitoring-friendly userProperties, and the KQL cheat-sheet is written out so operators can drop the queries into Log Analytics on day one.

ParameterCatalog

Single source of truth for every ADF parameter name used by the generated artifacts. Centralising the names means linked-service builders, dataset builders, copy-activity builders, the orchestrator's adf-parameters.template.json, and the unit tests all reference identical strings — typos cause test failures, not silent runtime errors.

ParameterCatalog.ParameterDefinition

Typed descriptor for a single ADF parameter: the Name surfaced in generated artifact JSON, the ADF Type token (e.g. "string"), a human-readable Description, and an optional DefaultValue baked into the parameter block at generation time.

SqlIdentifierEscaper

Bracket-quotes T-SQL identifiers using SQL Server rules: [name], with any literal ] doubled to ]]. Centralised so #145 row-count validation queries (and any future generated T-SQL) cannot be tricked by an assessment-supplied identifier that contains ].

SqlLiteralEscaper

Escapes T-SQL string literals by doubling single quotes (' → ''). Used by #147 incremental queries that bake the mapping key as a literal into a sqlReaderQuery / Script body (the alternative would be ADF Lookup parameter binding, which Azure SQL Lookup does not support without switching to SqlServerStoredProcedure).

UserPropertiesBuilder

Builds the ADF activity-level userProperties list that flows into the UserProperties custom-dimension column on the ADFActivityRun Log Analytics table (when the diagnostic setting uses logAnalyticsDestinationType = "Dedicated"). Centralising the shape means every Copy activity in every pipeline emits the same monitoring contract.

ValidationActivityBuilder

Builds the (LookupSource, LookupTarget, IfCondition) activity triplet that performs row-count parity validation around a Copy activity (#145).

Pattern (per Microsoft Learn): LookupSrc → Copy → LookupTgt → IfCondition(expression) └ ifFalseActivities: [Fail]

Caller is responsible for chaining the Copy activity's dependsOn to the LookupSource so it doesn't start before the baseline count is captured.

ValidationActivityBuilder.ValidationTriplet

Triplet returned by Build(ContainerMapping, string, string, string, AdfNameRegistry, ValidationOptions). Caller adds these to the pipeline activity list and wires the Copy activity's dependsOn to LookupSourceName. Total activity contribution (including nested Fail inside IfCondition) is 4.

ValidationOptions

Row-count validation configuration (#145). Defaults are conservative: validation is enabled with strict equality and zero tolerance. Operators that expect drift (e.g. mid-load Cosmos writes, soft-deleted rows) can opt down to RowCountAtLeast with a tolerance budget.

WatermarkSchemaBuilder

Generates the idempotent T-SQL DDL for the per-mapping watermark table that the #147 incremental copy pattern reads from / writes to. The same DDL is emitted as a stand-alone .sql file under ADF/SQL/ and (when EnsureWatermarkTableAtRuntime is on) embedded inside an ADF Script activity at the start of every per-db pipeline so the pipeline self-bootstraps the table on first run.

Schema: [mappingKey] NVARCHAR(450) PRIMARY KEY — composite key chosen to fit SQL Server's 900-byte index limit (NVARCHAR(450) = 900 bytes). [lastTs] BIGINT NOT NULL DEFAULT 0 — Unix seconds; matches Cosmos _ts domain. [updatedUtc] DATETIME2(3) NOT NULL DEFAULT SYSUTCDATETIME() — audit column; not used by the runtime but useful for ops.
