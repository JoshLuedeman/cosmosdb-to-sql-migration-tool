Remarks

The detector keeps a fixed-length window of recent observed values per (pipeline, run, activity, metric) key. For each new observation it evaluates the value against the baseline formed by the prior values in the window (so a value never contaminates its own baseline), then appends it.

This type is stateful and not thread-safe. It is intended to be driven by a single streaming loop (the scoped CLI run). Do not share one instance across concurrent consumers. The fixed sample-count window is a heuristic and does not model elapsed time or migration phase changes; tune ZScoreThreshold and MinRelativeChange if false positives occur.
