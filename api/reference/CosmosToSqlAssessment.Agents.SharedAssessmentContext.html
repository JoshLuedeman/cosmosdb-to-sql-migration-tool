Thread-safe blackboard implementation of ISharedAssessmentContext. A single private lock guards every piece of mutable state (the four domain outputs, the message log, and the result log) so concurrently scheduled agents observe a consistent view.

Remarks

Domain outputs are write-once: each Set* method throws InvalidOperationException on a second write, which surfaces accidental concurrent double-production as a clear failure rather than silent corruption. Reads of the domain outputs return the committed reference (the underlying model objects are not mutated after being committed); reads of Messages and Results return immutable snapshots taken under the lock.
