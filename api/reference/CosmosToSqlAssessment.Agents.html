Namespace CosmosToSqlAssessment.Agents

Classes

AgentMessage

An immutable, timestamped log entry produced by an agent and appended to the shared Messages blackboard.

AgentOrchestrationOptions

Options that control how the AgentOrchestrator schedules and bounds an agentic run.

AgentOrchestrationResult

The immutable outcome of an AgentOrchestrator run. It exposes point-in-time snapshots only; the live, mutable SharedAssessmentContext is intentionally not surfaced so callers cannot retroactively change the projected result.

AgentOrchestrator

Coordinates the assessment agents over a single SharedAssessmentContext, scheduling them according to their declared Dependencies and the requested AgentExecutionMode, isolating per-agent failures, and finishing with the validation pass.

AgentResult

Immutable record of how a single agent run finished. Recorded on the SharedAssessmentContext so the orchestrator can isolate failures and the ValidatorAgent can reason about completeness.

AssessmentAgentBase

Base class for assessment agents that centralises run timing, structured logging, and the failure-isolation contract so concrete agents only implement their domain logic.

CosmosAnalyzerAgent

Agent that owns the Cosmos DB analysis domain. It wraps the existing CosmosDbAnalysisService (whose signatures are intentionally unchanged) and commits the resulting CosmosDbAnalysis to the shared context for downstream agents.

DataQualityAgent

Agent that owns the pre-migration data-quality domain. It wraps the existing DataQualityAnalysisService (unchanged) and commits the resulting DataQualityAnalysis to the shared context.

SharedAssessmentContext

Thread-safe blackboard implementation of ISharedAssessmentContext. A single private lock guards every piece of mutable state (the four domain outputs, the message log, and the result log) so concurrently scheduled agents observe a consistent view.

SqlPlannerAgent

Agent that owns the Azure SQL migration-planning domain. It wraps the existing SqlMigrationAssessmentService (unchanged) and commits the resulting SqlMigrationAssessment to the shared context.

ValidationFinding

A single observation produced by the ValidatorAgent while cross-checking agent outputs.

ValidationReport

The typed verdict produced by the ValidatorAgent after cross-checking the outputs of the other agents. It is the stable, structured surface downstream consumers (and the orchestrator/CLI) use to decide whether an agentic run succeeded.

ValidatorAgent

Agent that cross-checks the outputs produced by the other agents and emits a typed ValidationReport describing whether the run is complete and internally consistent.

Interfaces

IAssessmentAgent

A single-responsibility participant in an agentic assessment run. Each agent owns one AgentRole, reads any upstream outputs it needs from the shared context, performs its work, and commits its own output back to the context.

ISharedAssessmentContext

Thread-safe shared blackboard through which assessment agents communicate. Agents read the outputs produced by upstream agents, contribute their own domain output exactly once, append log messages, and record their run results. Implemented by SharedAssessmentContext.

Enums

AgentExecutionMode

AgentMessageLevel

Severity of an AgentMessage recorded on a SharedAssessmentContext.

AgentRole

Identifies the assessment domain an IAssessmentAgent owns and the kind of output it contributes to a SharedAssessmentContext.

AgentRunStatus

Terminal outcome of a single IAssessmentAgent run.

ValidationFindingCategory

Classifies a ValidationFinding by the kind of problem it represents, so consumers can reason about which findings affect run completeness versus cross-output consistency versus pure diagnostics.
